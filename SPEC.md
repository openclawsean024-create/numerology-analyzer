# 生命靈數分析器 — 規格計劃書 v1.0

## 1. 問題陳述
Notion簡報稱，用戶輸入名字，自動計算命理，並收到一份完整的報告，涵蓋五個核心數字：先天數字、命運數字、靈魂數字、生日數字和個性數字，以及性格特徵、職業方向、愛情建議和年度週期。 Vercel 的即時標題是命理學分析器 |探索您的號碼代碼。

大多數數字命理學工具僅計算一個生命路徑數字並傳回固定的通用文字。使用者很快就能完成，但不知道如何將洞察力應用到工作、人際關係或年度計劃中。其痛點是計算規則不清晰、中文名稱支援弱、五數無法整合讀取、視覺化程度低、無法保存、共享性低。

## 2. 解決方案
建立快速、精美、可分享的命理報告。使用者輸入中文或英文姓名、生日和可選的顯示名稱。系統計算五個核心編號，在適當的時候保留主編號 11、22 和 33，並在詳細章節之前顯示儀表板。快速模式只能使用姓名，而完整模式則需要命運的生日、生日和年度週期編號。

該報告透過優勢、盲點、最佳環境、職業風格、愛情模式和年度關注點來解釋每個數字。這種體驗應該像一場高級人格測驗：簡單的輸入、即時的回饋、漂亮的數位卡、溫暖的語言，並且沒有絕對的預言。人工智慧可以根據確定性設定檔個性化措辭，但它無法更改計算出的數字。

## 3. 功能清單
MVP功能包括輸入驗證、中英文名處理、生日解析、核心號碼引擎、主號碼規則、報告頁面、能源雷達、年度週期時間軸、分享圖片、短連結、PDF或Markdown匯出、完整報告付費解鎖。免費版本應在一分鐘內產生有用的摘要。

高級功能包括關係相容性、名稱變更模擬、品牌或公司名稱分析、每月提醒、人工智慧跟進問題、報告歷史記錄、多語言內容和可嵌入小部件。管理員必須編輯數位解釋副本、常見問題、共享模板和定價，而無需更改程式碼。每個頁面必須註明內容是為了娛樂和自我探索，而不是醫療、投資、法律或重大人生決策建議。

## 4. 技術棧
使用 Next.js 或帶有 TypeScript、Tailwind CSS 和 Vercel 的 React 單頁應用程式。將計算規則放在lib/numerology.ts中，並輸出標準的NumerologyProfile JSON。報告範本將按數字查找文字與交叉數位合成結合。如果新增帳戶，則在 Supabase Postgres 中儲存設定檔、報告、共享連結、付款和內容區塊。

PDF可以使用react-pdf或print樣式；共享卡可以使用 Vercel OG Image 或 html-to-image。 Stripe Checkout 處理付款，Auth.js 或 Supabase Auth 處理帳戶。分析追蹤計算開始、報告完成、共享點擊和付費解鎖。測驗內容涵蓋中文姓名、英文姓名、空格、生日格式、主號碼保存、縮減規則、年度週期計算等。僅儲存必要的個人資料並提供刪除。

## 5. 完成標準 DoD
當使用者可以在 60 秒內完成輸入並查看所有五個核心編號摘要時，MVP 就完成了。完整的報告必須包括至少六章和一個年度週期部分。每個數字計算都有單元測試和可讀的解釋。行動分享卡不得裁剪中文文本，PDF輸出必須正確呈現中文，匿名使用者可以分享，登入使用者可以儲存歷史記錄。

品質標準包括 Lighthouse 效能高於 90、SE​​O 高於 90、報告產生 p95 在沒有 AI 的情況下低於 5 秒、錯誤率低於 1%、如果 AI 失敗則回退固定副本、隱私權政策、退款政策和免責聲明。 1 至 9 以及 11、22、33 的內容必須完整且不含基於恐懼或歧視性的語言。至少 25 名測試人員應該完成一份報告，其中 70% 的人表示報告很容易理解。

## 6. 風險與決策
風險一是僅輸入名稱無法計算每個核心編號。決策：明確區分快速模式和完整模式，並說明命運需要生日、生日、年度循環。風險之二是中文名轉換的分歧。決策：發布 MVP 的選定規則，並保留拼音、筆劃數或自訂規則作為後續選項。

風險三是模板化輸出。決定：綜合五個數字和年度週期，而不是只顯示孤立的段落。風險四是使用者對神秘內容的敏感度。決策：將其定位為娛樂和自我探索，避免可怕的預測。風險五是支付價值薄弱。決策：免費展示核心摘要；付費可解鎖完整報告、PDF、年度規劃、名稱模擬和相容性。

發布優先考慮計算信任和共享。第一個版本應在每個數字旁邊顯示簡單的計算說明，然後引導使用者儲存或共用報表。如果使用者對結果有疑問，請在新增更多AI副本或進階神秘功能之前提高規則透明度。
